Formulaire de contact

Préparer ton message

Cette version GitHub Pages ne possède pas encore de serveur d’envoi. Le bouton prépare un courriel avec ton message. Si ton application de courriel ne s’ouvre pas, copie le message et écris à contact@lepivotdor.ca.

Réponse

Pour recevoir une réponse utile

  • Écris avec le même courriel que celui utilisé pour l’achat.
  • Indique le nom exact de la formation.
  • Ajoute la date approximative du paiement si nécessaire.
  • Décris le problème en une phrase claire.

Sécurité

Ce qu’il ne faut pas envoyer

  • Numéro complet de carte bancaire.
  • Mot de passe personnel.
  • Clé API ou clé Stripe secrète.
  • Informations très sensibles ou privées.

FAQ contact

Questions fréquentes

Est-ce que le formulaire envoie vraiment le message?

Sur GitHub Pages, il n’y a pas de serveur d’envoi. Le formulaire prépare un courriel dans ton application de messagerie.

Pourquoi ne pas mettre mon mot de passe dans le message?

Parce qu’un mot de passe ou une information bancaire ne doit jamais être envoyé dans un formulaire de contact simple.

Que faire si mon accès ne fonctionne pas?

Va sur la page support et prépare ton courriel d’achat, le nom de la formation et le reçu Stripe si disponible.

Est-ce que je peux demander un remboursement ici?

Tu peux poser une question, mais les formations numériques sont encadrées par les conditions et la politique de vente finale.

Résumé : cette page sert à préparer une demande claire. Pour les problèmes d’accès, utilise d’abord la page support. Pour les questions légales ou commerciales, lis les conditions avant d’envoyer ta demande.
