Les dossiers ne sont pas créés automatiquement par le HTML. Tu dois les créer dans GitHub avec au moins un fichier à l’intérieur.

Tests GitHub avant publication

Avant de partager le lien, teste ces cinq points.

Cette section réduit les erreurs 404, les mauvais noms de fichiers et les oublis de base.

1. AccueilOuvre index.html et vérifie que les boutons rapides ne défilent pas horizontalement.
2. FormationTeste formation.html depuis le bouton Formation prioritaire.
3. SupportTeste support.html depuis le menu et depuis le footer.
4. LogoVérifie assets/logo/logo.png ou logo.png à la racine.
5. SécuritéNe mets aucun contenu payant complet ou donnée client dans GitHub public.
Important : une page cachée sur GitHub Pages n’est pas une vraie protection. Ne mets jamais le contenu complet payant, des données clients, des clés API, des liens privés sensibles ou des informations Stripe confidentielles dans ce dépôt public.
