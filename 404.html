Corrections fréquentes

À vérifier dans GitHub

  • Le fichier existe vraiment à la racine du dépôt.
  • Le nom est exactement pareil : minuscules, tirets, accents, extension.
  • Le fichier se termine bien par .html.
  • Le commit a été fait dans la bonne branche publiée par GitHub Pages.
  • Le lien interne pointe vers le bon fichier.
  • Le dossier ou le fichier n’a pas été nommé avec un espace invisible.
Exemple classique : conditionalite.html ne mène pas à conditions.html. Une seule lettre différente peut créer une erreur 404.
